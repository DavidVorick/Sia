How can I install Sia?
Sia is being actively developed, and is not yet ready for widespread use. However, an unstable alpha version is available for download. Currently, Sia is available for Linux only. Binaries for other platforms will be available during the beta.
Please read the "What's not in the alpha?" section below before downloading.

GitHub (recommended):
  • Clone the repo:
    git clone http://github.com/NebulousLabs/Sia && cd Sia
  • Install dependencies:
    make dependencies
  • Build Sia: 
    make
  • Client and server binaries are now installed in Sia/bin/

Direct download:
  • Select the appropriate platform:
    • Linux x86_64
  • Extract the tarball: 
    tar -xvjf sia-v0.0.1-linux-x86_64.tar.bz2
  • Client and server binaries can be found in the extracted folder.

You will need libsodium installed. Running 'make dependencies' should install libsodium automatically, however you will need autotools and you will need to provide your root password. For more help with libsodium, please read here. (Since Go compiles binaries statically, this should be compatible across most Linux distributions. Please let us know if you encounter problems.)

How do I use Sia?
To create a new server, run the "server" binary and enter "j" to join a quorum, or "e" to establish a new quorum. You will see scrolling output as the server compiles blocks and receives input. After you join a quorum, the server will stop accepting input, so we recommend running the server in its own window.

To connect to a quorum as a client, run the "client" binary. The client will attempt to auto-connect to a quorum running on localhost:9988. Enter "h" to view a list of client commands.

What's not in the alpha?
Sia is a complex project with lots of features that are not represented in the alpha. You should view this release as a "proof of concept" instead of a usable product.
The following features are not available in this release:
  • A fully trust-free quorum: hosts are assumed to be honest.
  • Bankruptcy: wallets are not deleted when they reach a balance of 0.
  • Self-repairing files: sectors are not repaired when hosts leave the network.
  • Large files: uploading a large file will cause hosts to desynchronize.
  • Automatic port negotiation: cross-router connections must ensure ports are forwarded.
  • Meta quorums: communication is restricted to a single quorum.
These features are considered high-priority for the next release.
